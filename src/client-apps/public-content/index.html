ASMail (Authenticated Secure Mail)

To run this demo application, do the following service setup steps:

  • Create some test user identities. These are used in authentication steps, which are done both by server, and by client sides. Identity verification should be done in a non-tracking fashion. Mozilla's BrowserID accomplishes just that within a browser. We liberate from browser's origins best protocol's aspects, placing them into our MailerId protocol, which is used for identities.
  • Create account(s) in ASMail server for recipient(s). ASMail server only serves a role of an always reachable inbox, and a source of recipient's initial public key (in the absence of an off-line introduction). Sender's client talks directly with recipient's server. Therefore, ASMail server only knows that its user gets messages, but cannot know to whom user sends messages.
  • Create account(s) in 3NStorage server for all users. All messages are encrypted with short-lived keys. Each sender-recipient pair has its own keys. This creates a requirement to have a key management based on some storage. For some implementations, such storage can/must be local. But for an everyday common use of multiple synchronized devices, we need an online storage service, which adheres to the principles of 3N (when there is No unencrypted data, when there is No unnecessary metadata, then there is Nothing to steal from the server).
